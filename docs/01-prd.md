# 《产品需求文档 PRD》—— 化工包装袋报价系统

> 版本：v0.1（Phase 1 草案）
> 日期：2026-04-24
> 关联文档：`docs/02-database-design.md`、`docs/03-api-spec.md`

---

## 1. 产品定位

一款**移动端优先、可在微信内秒开**的化工包装袋报价 H5 应用。让采购客户跳过「电话/微信问价 → 销售翻表查价 → 回报」的传统链路，直接在手机上**三步选完参数即得到报价**；同时给企业销售/运营提供 PC 端后台来统一维护「型号 / 尺寸 / 材质 / 价格」。

### 1.1 价值主张

| 角色 | 痛点 | 本系统解法 |
|---|---|---|
| 采购客户 | 询价响应慢、规格记不住、价格不透明 | 微信里一条链接 → 三步选完 → 立刻出价 |
| 销售 | 反复回答相似问题、报价口径不一致 | 转发链接代替口播；价格在后台统一维护 |
| 运营/老板 | Excel 报价表分散、调价后销售用旧价 | 价格中心化、调价即时生效、可软下架 |

---

## 2. 用户角色

| 角色 | 端 | 是否需登录 | 主要场景 |
|---|---|---|---|
| 采购客户（C 端） | 移动端 H5（微信内 / 浏览器） | **否**（默认不强制留资） | 选型号→选尺寸→选材质→看价 |
| 销售 / 运营（STAFF） | PC 后台 | 是 | 维护基础库、维护价格、复制报价链接 |
| 管理员（ADMIN） | PC 后台 | 是 | 同上 + 管理后台账号 |

> 角色权限粒度先做**两级**（ADMIN / STAFF），ADMIN 比 STAFF 多一个「用户管理」入口。其他业务操作权限相同。

---

## 3. 核心用户故事

### 3.1 C 端 —— 采购客户

| ID | 故事 | 验收标准 |
|---|---|---|
| C-1 | 作为客户，我希望微信里点开链接就能用，不用注册不用安装 | 首屏 < 2s 可交互；不出现登录拦截 |
| C-2 | 作为客户，我希望先选「型号」，下拉里只列出在售型号 | `isActive=true` 的 Model 才出现 |
| C-3 | 作为客户，选完型号后，「尺寸」下拉**只显示该型号有报价的尺寸** | 联动查询命中 Price 表 |
| C-4 | 作为客户，选完尺寸后，「材质」下拉**只显示该 (型号+尺寸) 有报价的材质** | 同上 |
| C-5 | 作为客户，三项选完后，立刻显示**单价**与备注（如「不含税」） | 无需点「查询」按钮，自动触发 |
| C-6 | 作为客户，我希望输入「数量」后立刻看到「总价 = 单价 × 数量」 | 数量为空时只显示单价 |
| C-7 | 作为客户，当我的数量低于 MOQ（最小起订量）时，要被明显提示 | 显示红字「最小起订量为 X 条」 |
| C-8 | 作为客户，我希望一键复制报价文本，方便发给同事 / 老板 | 复制内容含：型号、尺寸、材质、单价、数量、总价、时间戳 |
| C-9 | 作为客户，我希望切换参数时已选项不被无端清空 | 切换上层后，保留下层「同名/相同 ID」的选择，否则清空 |
| C-10 | 作为客户，发现没有匹配组合时要有友好提示 | 显示「该组合暂未配置报价，请联系销售」 |

### 3.2 B 端 —— 销售 / 运营

| ID | 故事 | 验收标准 |
|---|---|---|
| B-1 | 作为运营，我用账号密码登录后台 | 登录态通过 cookie/session 维持，刷新不掉线 |
| B-2 | 作为运营，我可以增删改查【型号】 | 表单含 code、name、description、isActive、sortOrder |
| B-3 | 作为运营，我可以增删改查【尺寸】 | 表单含 name、width、height、gusset、unit |
| B-4 | 作为运营，我可以增删改查【材质】 | 表单含 name、description |
| B-5 | 作为运营，我可以**新增价格**：选 (型号, 尺寸, 材质) 三元组 + 录入单价 | 三元组重复时报错并提示已有价格 |
| B-6 | 作为运营，我可以**按型号 / 尺寸 / 材质筛选**价格列表 | 列表支持组合筛选 + 分页 |
| B-7 | 作为运营，我可以**批量软下架**：勾选多条后切换 isActive | 软下架后 C 端立即不可见 |
| B-8 | 作为运营，删除型号/尺寸/材质时要有强提示「将级联删除 N 条价格」 | 弹窗二次确认 |
| B-9 | 作为管理员，我可以新建/禁用其他后台账号 | 仅 ADMIN 角色可见此入口 |

> **暂不在 v1 范围**：Excel 批量导入价格、操作审计日志、客户询价记录、价格历史曲线。这些列入 v2 候选。

## 4. 页面结构树（信息架构）

```
chemical-bag-quote.example.com
│
├─ C 端（移动端优先，无登录）
│   └─ /                       报价计算器（核心单页）
│       ├─ 顶栏          品牌名 + 客服电话/微信
│       ├─ 型号选择       下拉选择器
│       ├─ 尺寸选择       下拉选择器（依赖型号）
│       ├─ 材质选择       下拉选择器（依赖型号+尺寸）
│       ├─ 数量输入       数字框（可选，留空时只显示单价）
│       ├─ 报价结果卡    单价 / 总价 / 备注 / MOQ 提示
│       └─ 操作区        「复制报价」「联系销售」按钮
│
└─ B 端（PC 优先，需登录）
    ├─ /admin/login            登录
    ├─ /admin                  仪表盘（默认跳转到 /admin/prices）
    ├─ /admin/models           型号管理（列表 + 新增/编辑弹窗）
    ├─ /admin/sizes            尺寸管理（同上结构）
    ├─ /admin/materials        材质管理（同上结构）
    ├─ /admin/prices           价格管理（核心：筛选 + 列表 + 新增/编辑/批量操作）
    └─ /admin/users            后台账号管理（仅 ADMIN 可见）
```

## 5. 关键交互流程

### 5.1 C 端报价流程（Happy Path）

```
进入页面
   ↓
[GET /api/public/models]
   ↓
用户选定 Model
   ↓
[GET /api/public/sizes?modelId=...]
   ↓
用户选定 Size
   ↓
[GET /api/public/materials?modelId=...&sizeId=...]
   ↓
用户选定 Material
   ↓
[GET /api/public/quote?modelId=&sizeId=&materialId=]
   ↓
显示单价；用户输入数量 → 前端本地计算总价
   ↓
（可选）用户点「复制报价」→ 写入剪贴板
```

### 5.2 B 端价格新增流程

```
登录 → 进入 /admin/prices → 点「新增价格」
   ↓
弹窗内三个下拉（Model / Size / Material，全量加载）
   ↓
填 unitPrice、moq、remark → 提交
   ↓
[POST /api/admin/prices]
   ├─ 三元组已存在 → 422，前端提示「已存在，请直接编辑」
   └─ 成功 → 列表刷新
```

## 6. 非功能需求

| 维度 | 目标 |
|---|---|
| 性能 | C 端首屏 LCP < 2s（4G 网络）；联动接口 P95 < 300ms |
| 兼容 | iOS Safari 14+ / Android Chrome 90+ / 微信内置浏览器 |
| 响应式 | C 端 320~768px 优先；B 端 1280px+ 优先 |
| 安全 | 后台密码 bcrypt；后台接口必须校验 session；C 端只读、不暴露 admin 路由 |
| 可观测 | 至少打印 API 错误日志到 server console；后续接 Sentry |
| 部署 | 支持 Vercel 一键部署 + Docker 自托管两种方式（Phase 5 输出指南） |

## 7. 设计风格指引

- **C 端**：Shadcn/ui 默认主题，单列布局，按钮 ≥ 44px 高度（拇指友好），主色建议工业感的深蓝 / 墨绿。
- **B 端**：Shadcn/ui 表格 + Dialog 标准模式，避免花哨动画，强调列表密度与操作效率。
- 字体：系统默认（中文 PingFang / Microsoft YaHei）。

---

## 8. 范围边界（明确不做）

| 不做 | 原因 |
|---|---|
| 在线下单 / 在线支付 | 业务仍需销售人工对接订单与合同 |
| 客户账号体系 | 强制留资会显著降低 C 端使用率 |
| 多语言 | v1 仅中文 |
| 移动端原生 App / 小程序 | 与「H5 优先」决策一致，避免审核与多端维护成本 |

## 9. 验收 Demo 清单

v1 上线前，应能演示以下 5 个场景：

1. 在手机微信内打开链接，30 秒内完成一次完整报价并复制文本。
2. 故意选一个不存在的组合，看到友好提示。
3. 在后台新增一个型号 + 尺寸 + 材质 + 价格，C 端立刻可见。
4. 在后台把某条价格下架，C 端联动选项立刻消失。
5. 删除一个型号，弹出「将级联删除 N 条价格」二次确认。
